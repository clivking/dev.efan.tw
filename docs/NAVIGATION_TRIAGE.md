# Navigation Triage

## Goal

Classify the current visible navigation into three buckets:

- must keep
- can move or demote
- can review later

This is about public-facing priority, not whether the code exists.

## Must Keep

These entries are core to the current website story and conversion path.

### Header

- `/products`
- `/products/category/[slug]`
- `/services/*`
- `/guides`
- `/about`
- `/contact`
- `/quote-request`

Reason:

- products, services, and quote request are the main revenue path
- guides support discovery and SEO
- about and contact support trust and conversion

### Mobile menu

- products section
- services section
- guides entry
- about
- contact
- quote request CTA

Reason:

- mobile users still need the same core path as desktop users

### Footer

- company info
- phone and email
- service links
- guides entry
- quote request CTA
- privacy
- terms

Reason:

- these are core business and compliance links

## Can Move Or Demote

These items can stay on the site, but they do not need the same prominence as the core path.

### Resource center support links

- `/tools/cctv-storage-calculator`
- `/support/downloads`

Suggested handling:

- keep under guides/resources/support pages
- do not let them compete visually with products or quote request

### Footer location SEO links

- `/locations/*`

Suggested handling:

- keep in footer or dedicated SEO landing area
- do not promote into top navigation unless there is a strong business reason

### About submenu detail

- `/about/clients`

Suggested handling:

- keep under about
- does not need primary CTA-level prominence

## Review Later

These are visible now but are not clearly first-line public navigation priorities.

### Portal

- `/portal`

Current position:

- shown in the resource center
- shown in mobile resource list

Why review later:

- useful for existing customers
- not part of the main new visitor conversion path
- may be better placed as a smaller utility link instead of a featured resource tile

## Suggested Public Nav Order

If we later simplify the navigation, this is the recommended priority order:

1. products
2. services
3. guides
4. about
5. contact
6. quote request CTA

Utility or secondary surface:

- downloads
- calculator
- portal
- location pages
- client showcase subpage

## Current Recommendation

For the next UI cleanup pass:

- keep products, services, guides, about, contact, and quote request exactly visible
- keep downloads, calculator, and location links but treat them as secondary
- keep portal available, but consider reducing its prominence later instead of removing it
